# 完整交互流程说明 - PDF 对话历史功能

## 🎯 用户交互流程

你想要的完整交互流程已经实现：

### 流程 1️⃣: 上传 PDF 并进行对话

```
用户上传 PDF
    ↓
上传 API 保存 PDF 信息到数据库
    ↓
创建对话记录 (pdf_conversations)
    ↓
用户提问
    ↓
聊天 API 处理问题
    ↓
AI 生成回答
    ↓
保存对话消息到数据库 (conversation_messages)
    ├─ 保存用户问题
    └─ 保存 AI 回答
    ↓
用户看到 AI 回答
```

### 流程 2️⃣: 刷新页面后查看历史

```
用户刷新页面
    ↓
PDFList 组件加载
    ↓
调用 GET /api/pdfs/list
    ↓
从数据库获取用户的所有 PDF
    ↓
显示 PDF 列表
    ├─ 文件名
    ├─ 文件大小
    ├─ 页数
    ├─ 对话数量 ← 从 pdf_conversations 表获取
    └─ 最后对话时间
    ↓
用户可以看到之前上传的 PDF
```

### 流程 3️⃣: 点开 PDF 查看历史对话

```
用户点击 PDF 列表中的某个 PDF
    ↓
activePdfId 更新
    ↓
ConversationHistory 组件加载
    ↓
调用 GET /api/pdfs/{id}/conversations
    ↓
从数据库获取该 PDF 的所有对话消息
    ↓
显示对话历史
    ├─ 用户问题 (role: 'user')
    ├─ AI 回答 (role: 'assistant')
    ├─ 消息时间戳
    ├─ Token 数量
    └─ 处理时间
    ↓
用户可以看到完整的对话历史
```

### 流程 4️⃣: 继续对话

```
用户在聊天框输入新问题
    ↓
用户点击发送
    ↓
调用 POST /api/chat
    ↓
聊天 API 处理问题
    ↓
AI 生成回答
    ↓
保存新的对话消息到数据库
    ├─ 保存用户新问题
    └─ 保存 AI 新回答
    ↓
用户看到新的 AI 回答
    ↓
对话历史自动更新
```

## 📊 数据库存储结构

### user_pdfs 表 (PDF 文件信息)
```
id: UUID
user_id: UUID (用户ID)
filename: TEXT (文件名)
file_size: INTEGER (文件大小)
storage_path: TEXT (存储路径)
page_count: INTEGER (页数)
text_summary: TEXT (文本摘要)
parse_status: TEXT (解析状态)
created_at: TIMESTAMP (创建时间)
updated_at: TIMESTAMP (更新时间)
```

### pdf_conversations 表 (对话记录)
```
id: UUID
pdf_id: UUID (关联的 PDF)
user_id: UUID (用户ID)
message_count: INTEGER (消息数量)
last_message_at: TIMESTAMP (最后消息时间)
created_at: TIMESTAMP (创建时间)
updated_at: TIMESTAMP (更新时间)
```

### conversation_messages 表 (对话消息)
```
id: UUID
conversation_id: UUID (关联的对话)
pdf_id: UUID (关联的 PDF)
user_id: UUID (用户ID)
role: TEXT ('user' 或 'assistant')
content: TEXT (消息内容)
tokens: INTEGER (token 数量)
processing_time: INTEGER (处理时间)
created_at: TIMESTAMP (创建时间)
```

## 🔄 API 调用时序

### 上传 PDF 时
```
POST /api/upload
    ↓
保存 PDF 信息到 user_pdfs
    ↓
创建对话记录到 pdf_conversations
    ↓
返回 PDF ID
```

### 发送消息时
```
POST /api/chat
    ↓
验证用户和配额
    ↓
搜索相关文档
    ↓
调用 AI 模型
    ↓
流式返回 AI 回答
    ↓
保存对话消息到 conversation_messages
    ├─ 保存用户问题
    └─ 保存 AI 回答
    ↓
更新 pdf_conversations 的统计信息
```

### 刷新页面时
```
GET /api/pdfs/list
    ↓
查询 user_pdfs 表
    ↓
左连接 pdf_conversations 获取对话统计
    ↓
返回 PDF 列表
```

### 查看对话历史时
```
GET /api/pdfs/{id}/conversations
    ↓
验证用户权限
    ↓
查询 conversation_messages 表
    ↓
按时间排序
    ↓
返回对话消息
```

## 💾 数据持久化流程

### 第 1 步: 上传 PDF
```
用户上传 PDF
    ↓
上传 API 调用 savePDFInfo()
    ↓
INSERT INTO user_pdfs (
  id, user_id, filename, file_size, storage_path, parse_status
)
    ↓
上传 API 调用 createOrGetConversation()
    ↓
INSERT INTO pdf_conversations (
  id, pdf_id, user_id, message_count, last_message_at
)
    ↓
PDF 和对话记录已保存到数据库
```

### 第 2 步: 发送消息
```
用户发送消息
    ↓
聊天 API 调用 saveConversationExchange()
    ↓
INSERT INTO conversation_messages (
  id, conversation_id, pdf_id, user_id, role, content, tokens, processing_time
) VALUES (
  uuid, conv_id, pdf_id, user_id, 'user', question, null, null
)
    ↓
INSERT INTO conversation_messages (
  id, conversation_id, pdf_id, user_id, role, content, tokens, processing_time
) VALUES (
  uuid, conv_id, pdf_id, user_id, 'assistant', response, token_count, processing_time
)
    ↓
触发器自动更新 pdf_conversations:
  UPDATE pdf_conversations
  SET message_count = message_count + 2,
      last_message_at = NOW()
  WHERE id = conv_id
    ↓
对话消息已保存到数据库
```

### 第 3 步: 刷新页面
```
用户刷新页面
    ↓
PDFList 组件调用 getPDFList()
    ↓
SELECT p.*, c.message_count, c.last_message_at
FROM user_pdfs p
LEFT JOIN pdf_conversations c ON p.id = c.pdf_id
WHERE p.user_id = current_user_id
    ↓
返回 PDF 列表（包含对话统计）
    ↓
前端显示 PDF 列表
```

### 第 4 步: 查看对话历史
```
用户点击 PDF
    ↓
ConversationHistory 组件调用 getConversationHistory()
    ↓
SELECT * FROM conversation_messages
WHERE pdf_id = selected_pdf_id
AND user_id = current_user_id
ORDER BY created_at ASC
    ↓
返回对话消息列表
    ↓
前端显示对话历史
```

## 🔐 数据隔离和安全

### RLS 策略 (行级安全)
```
user_pdfs:
  - 用户只能看到自己的 PDF
  - SELECT: WHERE user_id = auth.uid()
  - INSERT: WHERE user_id = auth.uid()

pdf_conversations:
  - 用户只能看到自己的对话
  - SELECT: WHERE user_id = auth.uid()
  - INSERT: WHERE user_id = auth.uid()

conversation_messages:
  - 用户只能看到自己的消息
  - SELECT: WHERE user_id = auth.uid()
  - INSERT: WHERE user_id = auth.uid()
```

### 权限验证
```
所有 API 端点都验证：
1. 用户是否已认证
2. 用户是否拥有该 PDF
3. 用户是否拥有该对话

例如：
GET /api/pdfs/{id}/conversations
  ↓
验证用户已认证
  ↓
验证 PDF 属于该用户
  ↓
返回对话消息
```
