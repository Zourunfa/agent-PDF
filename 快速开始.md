# 快速开始 - PDF 对话历史功能

## 🚀 5 分钟快速开始

### 第 1 步: 确保数据库迁移已完成

```bash
# 运行数据库迁移
supabase migration up
```

### 第 2 步: 启动应用

```bash
# 启动开发服务器
npm run dev
```

### 第 3 步: 打开应用

1. 打开浏览器
2. 访问 `http://localhost:3000`
3. 登录你的账户

### 第 4 步: 上传 PDF 并进行对话

1. 在左侧面板点击"上传 PDF"
2. 选择一个 PDF 文件
3. 等待上传完成
4. 在右侧聊天框输入问题
5. 点击发送
6. 等待 AI 回答

### 第 5 步: 验证功能

1. **刷新页面** - 查看 PDF 列表是否仍然存在
2. **点击 PDF** - 查看对话历史是否显示
3. **继续对话** - 输入新问题并发送
4. **查看历史** - 点击"历史"标签页查看所有对话

## 🔍 如何验证

### 验证 1: 查看 API 调用

1. 打开浏览器开发者工具 (F12)
2. 切换到 Network 标签页
3. 刷新页面
4. 查看是否有 `/api/pdfs/list` 请求
5. 响应状态码应该是 200

### 验证 2: 查看对话历史

1. 点击 PDF 列表中的某个 PDF
2. 打开浏览器开发者工具 (F12)
3. 切换到 Network 标签页
4. 点击右侧面板的"历史"标签页
5. 查看是否有 `/api/pdfs/{id}/conversations` 请求
6. 响应状态码应该是 200

### 验证 3: 查看数据库

在 Supabase 仪表板执行：
```sql
-- 查看 PDF
SELECT * FROM user_pdfs WHERE user_id = 'your-user-id';

-- 查看对话记录
SELECT * FROM pdf_conversations WHERE user_id = 'your-user-id';

-- 查看对话消息
SELECT * FROM conversation_messages WHERE user_id = 'your-user-id';
```

## 🐛 常见问题

### Q: 为什么没有看到 PDF 列表？
A: 
1. 检查是否已上传 PDF
2. 检查浏览器控制台是否有错误
3. 检查 Network 标签页的 API 请求

### Q: 为什么对话历史为空？
A:
1. 检查是否选择了 PDF
2. 检查该 PDF 是否有对话记录
3. 检查浏览器控制台是否有错误

### Q: 为什么 API 返回错误？
A:
1. 检查是否已登录
2. 检查认证令牌是否有效
3. 检查数据库迁移是否完成
