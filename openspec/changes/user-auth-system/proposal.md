# 用户认证系统提案

## Why

当前 PDF AI Chat 应用缺乏用户认证和权限管理功能，存在以下问题：

1. **无用户隔离**：所有用户共享相同的数据存储，无法实现数据隔离和隐私保护
2. **无法追踪使用**：没有用户系统，无法追踪用户使用情况和统计数据
3. **无法商业化**：缺乏用户管理和配额控制，无法实现分级服务和商业化运营
4. **安全性问题**：无认证系统意味着任何人都可以无限制访问和使用资源
5. **用户体验受限**：用户无法保存自己的文档历史和对话记录

**为什么现在实现**：
- 产品已有核心功能，需要进入多用户阶段
- 用户增长需要用户管理和数据分析支持
- 未来商业化需要用户系统和配额管理基础

## What Changes

### 核心变化

1. **游客试用机制**：未登录用户可以免费体验3次PDF上传和AI对话
2. **用户认证系统**：完整的用户注册、登录、会话管理
3. **配额管理系统**：基于用户角色的资源限制（免费用户/高级用户）
4. **用户个人中心**：用户可以管理自己的资料、查看使用统计
5. **管理员后台**：管理员可以管理用户、设置配额、查看系统统计
6. **邮件通知系统**：注册验证、密码重置、配额警告等通知

### 技术架构变化

- 新增 Supabase PostgreSQL 数据库用于用户数据存储
- 新增 Supabase Auth 用于用户认证
- 新增邮件服务（Resend/SendGrid）用于通知
- 新增管理员后台页面
- 新增用户个人中心页面
- API 层增加认证中间件和配额检查

## Capabilities

### New Capabilities

- `guest-trial`: 游客试用机制（3次免费使用限制）
- `user-auth`: 用户认证系统（注册、登录、会话管理、密码重置）
- `quota-management`: 配额管理系统（基于角色的资源限制）
- `user-center`: 用户个人中心（资料管理、使用统计、密码修改）
- `admin-panel`: 管理员后台（用户管理、配额管理、系统统计）
- `email-notification`: 邮件通知系统（验证邮件、密码重置、配额警告）

### Modified Capabilities

- `pdf-upload`: 修改为需要认证和配额检查
- `ai-chat`: 修改为需要认证和配额检查，数据按用户隔离

## Impact

### 受影响的系统

1. **前端页面**
   - 新增：/login, /register, /user-center, /admin
   - 修改：首页增加登录/注册入口
   - 修改：所有需要认证的页面增加权限检查

2. **API 路由**
   - 新增：/api/auth/*, /api/user/*, /api/admin/*
   - 修改：/api/upload, /api/chat 增加认证和配额检查
   - 修改：所有PDF相关API增加用户数据隔离

3. **数据存储**
   - 新增：Supabase PostgreSQL 数据库
   - 新增：用户表、配额表、会话表、使用统计表
   - 修改：Pinecone 向量数据增加用户ID过滤
   - 修改：Redis 数据增加用户ID前缀

4. **依赖项**
   - 新增：@supabase/supabase-js, @supabase/auth-helpers-nextjs
   - 新增：resend 或 sendgrid（邮件服务）
   - 新增：bcrypt（密码加密）

5. **环境变量**
   - 新增：NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY
   - 新增：SUPABASE_SERVICE_ROLE_KEY
   - 新增：RESEND_API_KEY 或 SENDGRID_API_KEY

### 数据迁移

- 现有 Pinecone 数据需要标记为"系统数据"或迁移到测试账户
- 现有 Redis 缓存数据需要添加用户ID前缀

### 兼容性

- 向后兼容：游客模式仍然可以使用（3次限制）
- 已有用户：无需迁移（当前无用户系统）
- API：新增认证字段，但保持兼容性过渡期

## 实施阶段

本项目将分6个阶段实施，每个阶段都是独立可交付的里程碑：

### 阶段 1: 基础认证 + 免费试用 (1-2周)
- Supabase 集成和配置
- 用户表结构设计
- 游客计数机制（3次限制）
- 用户注册/登录流程
- 基础权限检查

### 阶段 2: 配额管理 + 使用追踪 (1-2周)
- 配额系统设计
- 使用统计追踪
- 配额检查中间件
- 使用统计展示

### 阶段 3: 用户个人中心 (1周)
- 个人资料编辑
- 密码修改
- 头像上传
- 使用历史查看

### 阶段 4: 邮件通知系统 (1周)
- 邮件服务集成
- 注册验证邮件
- 密码重置邮件
- 配额警告邮件

### 阶段 5: 管理员后台 (2周)
- 管理员权限系统
- 用户管理页面
- 配额管理页面
- 系统统计面板

### 阶段 6: 高级功能 (未来)
- 第三方登录（微信、Google、GitHub）
- 多设备会话管理
- API 频率限制
- 会员订阅系统

## 成功指标

- [ ] 游客可以成功体验3次PDF上传和AI对话
- [ ] 用户可以顺利注册和登录
- [ ] 用户数据完全隔离，无法访问其他用户数据
- [ ] 配额系统准确追踪和限制用户使用
- [ ] 管理员可以高效管理用户和配额
- [ ] 邮件通知系统稳定可靠
- [ ] 系统性能不受认证功能影响（响应时间 < 500ms）
