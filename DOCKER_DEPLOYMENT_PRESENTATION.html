💾 存储架构演变

☁️ Vercel Blob Storage

优点：

  • ✓ 无需配置
  • ✓ 自动扩展
  • ✓ 全球 CDN

缺点：

  • ✗ 数据在云端
  • ✗ 存储费用
  • ✗ Private 访问复杂

📁 Local File Storage

优点：

  • ✓ 数据本地化
  • ✓ 无额外费用
  • ✓ 完全控制
  • ✓ 支持离线

缺点：

  • ✗ 需要管理存储
  • ✗ 需要备份

🔄 智能切换

自动检测环境：

  • • STORAGE_PATH 设置 → 本地存储
  • • BLOB_READ_WRITE_TOKEN 设置 → Vercel Blob
  • • 代码零改动，透明切换
// 存储适配器自动选择 function getStorageType(): 'local' | 'vercel-blob' { if (process.env.STORAGE_PATH) { return 'local'; // Docker 自托管 } if (process.env.BLOB_READ_WRITE_TOKEN) { return 'vercel-blob'; // Vercel 部署 } return 'local'; // 默认本地 }

🔄 迁移步骤

  • 1
    环境准备

    安装 Docker 和 Docker Compose，准备服务器环境

  • 2
    配置环境变量

    复制 .env.docker.example 到 .env，填写必要的配置

  • 3
    启动服务

    运行 docker-compose up -d 启动所有容器

  • 4
    数据迁移

    从 Vercel Blob 下载 PDF 文件，导入数据库数据

  • 5
    验证部署

    检查所有服务状态，测试功能是否正常

⚡ 快速开始

# 1. 克隆代码 git clone https://github.com/yourusername/agent-pdf.git cd agent-pdf # 2. 配置环境变量 cp .env.docker.example .env nano .env # 编辑配置 # 3. 创建数据目录 mkdir -p data/{postgres,redis,qdrant,uploads} # 4. 启动服务 cd docker docker-compose up -d # 5. 查看日志 docker-compose logs -f # 6. 访问应用 open http://localhost

🔧 常见问题解决

容器无法启动

检查日志：docker-compose logs -f [service]
检查端口：sudo netstat -tulpn

数据库连接失败

检查网络：docker network inspect
验证环境变量：docker exec env | grep DATABASE

文件无法上传

检查权限：ls -la data/uploads/
修正权限：chown -R 1001:1001 data/

性能问题

查看资源：docker stats
增加限制：调整 docker-compose.yml

忘记密码

重置 DB：docker exec psql -c "ALTER USER..."
重置 Redis：编辑 docker-compose.yml

数据恢复

从备份恢复：docker cp backup.sql container:/tmp/
导入数据：psql -f /tmp/backup.sql
