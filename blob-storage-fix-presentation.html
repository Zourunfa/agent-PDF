📝 代码修改

新增 Blob 下载函数

// src/lib/storage/blob-storage.ts import { put, head, del, get } from '@vercel/blob'; export async function downloadPDFFromBlob(blobUrl: string): Promise<Buffer> { const result = await get(blobUrl, { access: 'private' }); // Read stream to buffer const reader = result.stream.getReader(); const chunks: Uint8Array[] = []; while (true) { const { done, value } = await reader.read(); if (done) break; chunks.push(value); } return Buffer.concat(chunks); }

🔄 修改对比

❌ 修改前（重定向）

if (storagePath && storagePath.startsWith('http')) { console.log('[PDF API] ☁️ Proxying download...'); const { downloadPDFFromBlob } = await import('@/lib/storage/blob-storage'); const fileBuffer = await downloadPDFFromBlob(storagePath); return new NextResponse(fileBuffer, { headers: { 'Content-Type': 'application/pdf', 'Cache-Control': 'public, max-age=3600', }, }); // ✅ 返回文件内容 }

⚙️ 技术细节

为什么 redirect 会 403？

Private Blob URL 需要认证 token，浏览器 redirect 请求时不带 BLOB_READ_WRITE_TOKEN，Vercel Blob Storage 拒绝未认证访问。

为什么代理下载可以？

服务端请求时自动使用环境变量中的 BLOB_READ_WRITE_TOKEN，@vercel/blob SDK 自动处理认证，服务端下载后返回给客户端。

性能考虑

服务端代理会增加延迟和服务器负载，但通过 Cache-Control 头可以减少重复下载。对于大文件，未来可考虑流式传输。

同样修复了 Parse API

在 src/app/api/parse/route.ts 中也添加了 Blob URL 下载逻辑，确保解析时能正确读取文件内容。
