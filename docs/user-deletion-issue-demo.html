完整的排查过程、问题根源和最佳实践

📋 问题描述

现象：管理员在后台删除用户后，用户仍然出现在用户列表中，看起来删除操作没有生效。

影响：管理员无法有效管理用户，可能导致数据不一致和安全问题。

问题日志

// 删除操作日志
[Admin Delete User] ✓ 用户数据清理完成
[Admin Delete User] 尝试从 auth.users 删除用户...
[Admin Delete User] auth.users 中未找到用户，可能已被删除

// 刷新列表后
[Admin Users] 从 user_profiles 获取到用户: [
  {
    id: 'b969a004-032c-4486-b695-dd68d725b593',
    email: '1804491927@qq.com',
    name: '阿锋'
  }
]

🔍 根本原因分析

主要原因：浏览器缓存 核心问题

问题机制：

  • Next.js 默认会缓存 API 响应以提高性能
  • 浏览器也会缓存 HTTP 响应（特别是 GET 请求）
  • 即使数据库中的数据已被删除，前端仍显示缓存中的旧数据

验证方法：清除浏览器缓存后，用户列表显示正确（空列表）

次要问题

问题 影响 状态
RPC 函数参数名不匹配 导致 RPC 删除失败，回退到逐个删除 已修复
删除逻辑不完整 user_profiles 中没有用户时，跳过删除 auth.users 已修复

✅ 解决方案

方案 1：配置 API 不使用缓存 推荐

在所有管理员 API 路由中添加：

// src/app/api/admin/users/route.ts
export const dynamic = 'force-dynamic';  // 禁用静态优化
export const revalidate = 0;              // 不重新验证缓存

// src/app/api/admin/users/[id]/route.ts
export const dynamic = 'force-dynamic';
export const revalidate = 0;

方案 2：前端添加缓存破坏参数

const fetchUsers = async () => {
  const cacheBuster = Date.now();  // 时间戳
  const response = await fetch(`/api/admin/users?t=${cacheBuster}`, {
    headers: {
      'Cache-Control': 'no-cache',
      'Pragma': 'no-cache',
    }
  });
  // ...
};

方案 3：删除成功后强制刷新

const handleDeleteUser = async (userId: string) => {
  // 删除用户
  await fetch(`/api/admin/users/${userId}`, {
    method: 'DELETE',
    cache: 'no-store',  // 禁用浏览器缓存
  });

  // 重新获取最新数据
  await fetchUsers();
};

方案 4：修复 RPC 函数参数问题

选择以下任一方式：

// 方式 A：修改函数定义（推荐）
CREATE OR REPLACE FUNCTION admin_delete_user_profile(
  user_id UUID  -- 改为 user_id
) ...

// 方式 B：修改代码调用
await supabase.rpc('admin_delete_user_profile', {
  target_user_id: userId  // 使用 target_user_id
});

💡 最佳实践

1. 管理员 API 配置

所有管理员相关的 API 都应该禁用缓存：

export const dynamic = 'force-dynamic';
export const revalidate = 0;

2. 删除操作流程

删除成功后应该重新获取最新数据：

// 1. 发送删除请求

3. 乐观删除 + 错误回滚

提升用户体验，立即反馈操作结果：

// 保存旧数据
const previousUsers = [...users];

// 乐观删除
setUsers(users.filter((u) => u.id !== userId));

try {
  await deleteUser(userId);
} catch (error) {
  // 失败时恢复
  setUsers(previousUsers);
}

4. 完整的删除逻辑

确保删除所有相关数据：

1. 删除 user_profiles
2. 删除 auth.users
3. 删除所有关联数据（PDF、对话、配额等）
