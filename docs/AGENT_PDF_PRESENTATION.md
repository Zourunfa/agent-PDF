## 什么是 AI Agent 应用

### 开场白

大家好！今天我想和大家分享一个激动人心的话题：AI Agent 应用。

在过去的一年里，我们见证了 ChatGPT 的爆火，也看到了各种 AI 对话应用的涌现。但你有没有想过，这些应用之间有什么本质区别？为什么有些 AI 只能简单回答问题，而有些却能帮你完成复杂的任务？

答案就在于：它们是否是真正的 AI Agent。

### 第一部分：三种 AI 应用的对比

让我先给大家展示三种不同层次的 AI 应用。

#### 1. Chatbot（聊天机器人）- 最基础的形态

```
用户：今天天气怎么样？
Chatbot：抱歉，我无法获取实时天气信息。
```

**特点：**
- 被动响应：只能回答问题，不能主动做事
- 单轮对话：没有上下文记忆
- 无工具调用：只依赖训练数据
- 容易产生幻觉：编造不存在的信息

这就像一个只会背书的学生，你问什么，它答什么，但它不会思考，也不会使用工具。

#### 2. RAG（检索增强生成）- 我们当前的状态

```
用户：这份 PDF 文档的主要内容是什么？
RAG：根据文档内容，主要讲述了...（基于真实文档回答）
```

**特点：**
- 被动响应：仍然是问答模式
- 多轮对话：有上下文记忆
- 单一工具：只能检索文档
- 准确性高：基于真实文档，不会胡编乱造

这就像一个会查资料的学生，它知道去哪里找答案，但仍然需要你告诉它要找什么。


#### 3. AI Agent（智能体）- 我们的升级目标

```
用户：帮我分析这份财报并生成投资建议。
Agent：好的，让我来帮你：
      1. 首先，我从文档中提取关键财务数据...
      2. 然后，我搜索行业平均水平进行对比...
      3. 接下来，我计算各项财务指标...
      4. 最后，基于分析结果，我的建议是...
```

**特点：**
- 主动规划：能够分解复杂任务
- 多步推理：一步步完成目标
- 多工具协作：搜索、计算、分析、生成
- 自主决策：知道何时该做什么

这就像一个真正的助手，你只需要告诉它目标，它会自己规划路径、使用工具、完成任务。

### 第二部分：Agent 的核心能力

现在让我深入讲解 AI Agent 的四大核心能力。

#### 能力 1：自主规划（Planning）

想象一下，你对 Agent 说："帮我准备明天的会议材料。"

一个普通的 AI 可能会说："请告诉我需要准备什么。"

但 Agent 会这样思考：

```
任务分解：
├─ 1. 了解会议主题和参与者
├─ 2. 搜索相关背景资料
├─ 3. 提取关键数据和图表
├─ 4. 生成会议议程
├─ 5. 准备演示文稿
└─ 6. 发送给参与者
```

这就是规划能力 - Agent 能够将一个模糊的目标分解成具体的、可执行的步骤。

#### 能力 2：工具调用（Tool Use）

Agent 不是孤立的，它可以调用各种工具来完成任务。

**工具箱示例：**

```typescript
const agentTools = [
  {
    name: "search_pdf",
    description: "从 PDF 文档中搜索信息",
    use_case: "需要查找文档内容时"

  {
    name: "calculator",
    description: "执行数学计算",
    use_case: "需要计算数据时"
  },
  {
    name: "web_search",
    description: "搜索互联网信息",
    use_case: "需要最新信息时"
  },
  {
    name: "generate_chart",
    description: "生成数据可视化图表",
    use_case: "需要展示数据时"
  },
  {

  }
];
```

就像一个工匠，Agent 知道什么时候该用锤子，什么时候该用螺丝刀。


#### 能力 3：推理循环（ReAct - Reasoning + Acting）

这是 Agent 最核心的能力 - ReAct 循环。让我用一个真实案例来说明。

**案例：分析一份合同的风险点**

```
用户输入：
"帮我分析这份合同的法律风险。"

Agent 的思考过程：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第 1 轮循环
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💭 Thought（思考）:
"要分析法律风险，我首先需要提取合同的关键条款。"

🔧 Action（行动）:
调用工具: search_pdf("违约条款 赔偿责任 终止条件")

👁️ Observation（观察）:
找到 3 个关键条款：
- 违约金条款：合同金额的 30%
- 赔偿责任：无上限
- 终止条件：单方面终止权

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第 2 轮循环
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💭 Thought（思考）:
"我发现了一些可能的风险点，但需要对比行业标准。"

🔧 Action（行动）:
调用工具: web_search("合同违约金标准 法律规定")

👁️ Observation（观察）:
根据《合同法》，违约金一般不超过实际损失的 30%，
但本合同的赔偿责任无上限，存在风险。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第 3 轮循环
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💭 Thought（思考）:
"现在我需要评估风险等级并生成建议。"

🔧 Action（行动）:
调用工具: risk_assessment({
  违约金: "30%",
  赔偿责任: "无上限",
  终止条件: "单方面"
})

👁️ Observation（观察）:
风险等级：高
主要风险：赔偿责任无上限可能导致巨额损失

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
反思与决策
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤔 Reflection（反思）:
"我已经完成了风险分析，可以生成最终报告了。"

✅ Decision（决策）:
任务完成，生成最终回答。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
最终输出
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 Final Answer:
根据分析，这份合同存在以下主要风险：

1. 高风险：赔偿责任无上限
   - 问题：可能导致不可预测的巨额损失
   - 建议：要求增加赔偿上限条款

2. 中风险：违约金 30%
   - 问题：处于法律允许的上限
   - 建议：可以接受，但需谨慎履约

3. 中风险：单方面终止权
   - 问题：对方可随时终止合同
   - 建议：增加终止通知期和补偿条款
```

看到了吗？这就是 ReAct 循环的威力 - Agent 不是一次性给出答案，而是像人类专家一样，一步步思考、行动、观察、反思，最终得出结论。


#### 能力 4：记忆系统（Memory）

Agent 需要记住三种类型的信息：

**1. 短期记忆（Working Memory）- 当前任务**

```
当前对话：
用户：分析这份财报
Agent：好的，我先提取数据...
用户：重点关注利润率
Agent：明白，我会重点分析利润率指标
      ↑ 记住了用户的补充要求
```

**2. 长期记忆（Long-term Memory）- 历史记录**

```
用户偏好：
- 喜欢详细的数据分析
- 偏好图表展示
- 关注风险提示

历史任务：
- 2025-03-01: 分析了 Q4 财报
- 2025-03-05: 审查了供应商合同
- 2025-03-07: 生成了投资建议报告
```

**3. 工作记忆（Episodic Memory）- 中间结果**

```
任务进度：
✅ 步骤 1: 提取财务数据（完成）
   结果：营收 100亿，利润 20亿
✅ 步骤 2: 计算利润率（完成）
   结果：20%
⏳ 步骤 3: 对比行业平均（进行中）
⬜ 步骤 4: 生成建议（待执行）
```

### 第四部分：从 RAG 到 Agent 的升级

现在让我展示我们项目的升级路径。

#### 当前状态：RAG 应用

```typescript
// 简单的问答流程
async function chat(pdfId: string, question: string) {
  // 1. 检索相关文档
  const docs = await searchDocs(pdfId, question);
  
  // 2. 生成回答
  const answer = await ai.chat({
    context: docs,
    question: question
  });
  
  return answer;
}
```

**局限性：**
- ❌ 只能回答问题，不能执行任务
- ❌ 不能处理复杂的多步骤需求
- ❌ 无法使用多种工具
- ❌ 缺乏规划和推理能力


#### 升级后：Agent 应用

```typescript
// 智能 Agent 流程
async function agentChat(pdfId: string, task: string) {
  // 1. 规划任务
  const plan = await planner.createPlan(task);
  // 返回: ["提取数据", "分析数据", "生成报告"]
  
  // 2. 执行计划（ReAct 循环）
  const results = [];
  for (const step of plan.steps) {
    // 2.1 思考
    const thought = await agent.think(step, results);
    console.log(`💭 思考: ${thought}`);
    
    // 2.2 选择工具
    const tool = await agent.selectTool(thought);
    console.log(`🔧 行动: 使用 ${tool.name}`);
    
    // 2.3 执行工具
    const result = await tool.execute();
    console.log(`👁️ 观察: ${result}`);
    results.push(result);
    
    // 2.4 反思
    const shouldContinue = await agent.reflect(results);
    console.log(`🤔 反思: ${shouldContinue ? '继续' : '完成'}`);
    
    if (!shouldContinue) break;
  }
  
  // 3. 生成最终回答
  const answer = await agent.synthesize(results);
  
  return answer;
}
```

**能力提升：**
- ✅ 可以执行复杂任务
- ✅ 支持多步骤推理
- ✅ 可以使用多种工具
- ✅ 具备规划和反思能力

### 第六部分：技术实现 - LangGraph

现在让我展示如何用 LangGraph 实现一个 Agent。

```typescript
import { StateGraph } from "@langchain/langgraph";

// 1. 定义状态
interface AgentState {
  messages: Message[];        // 对话历史
  plan: string[];            // 执行计划
  currentStep: number;       // 当前步骤
  toolResults: any[];        // 工具结果
  finalAnswer?: string;      // 最终答案
}

// 2. 定义节点函数

// 规划节点
async function plannerNode(state: AgentState) {
  const plan = await planner.createPlan(state.messages);
  return { ...state, plan };
}

// 执行节点
async function executorNode(state: AgentState) {
  const step = state.plan[state.currentStep];
  const tool = await selectTool(step);
  const result = await tool.execute();
  
  return {
    ...state,
    toolResults: [...state.toolResults, result],
    currentStep: state.currentStep + 1
  };
}

// 反思节点
async function reflectorNode(state: AgentState) {
  const isComplete = await evaluateCompletion(state);
  
  if (isComplete) {
    const answer = await synthesize(state.toolResults);
    return { ...state, finalAnswer: answer };
  }
  
  return state;
}

// 3. 构建状态图
const workflow = new StateGraph<AgentState>({
  channels: {
    messages: { value: (x, y) => x.concat(y) },
    plan: { value: (x, y) => y ?? x },
    currentStep: { value: (x, y) => y ?? x },
    toolResults: { value: (x, y) => x.concat(y) },
    finalAnswer: { value: (x, y) => y ?? x }
  }
});

// 4. 添加节点
workflow.addNode("planner", plannerNode);
workflow.addNode("executor", executorNode);
workflow.addNode("reflector", reflectorNode);

// 5. 添加边（定义流程）
workflow.addEdge("planner", "executor");
workflow.addEdge("executor", "reflector");

// 6. 添加条件边（决策点）
workflow.addConditionalEdges(
  "reflector",
  (state) => {
    // 如果有最终答案，结束
    if (state.finalAnswer) return "end";
    // 如果还有步骤，继续执行
    if (state.currentStep < state.plan.length) return "executor";
    // 否则重新规划
    return "planner";
  }
);

// 7. 设置入口
workflow.setEntryPoint("planner");

// 8. 编译成可执行的 Agent
const agent = workflow.compile();

// 9. 使用 Agent
const result = await agent.invoke({
  messages: [{ role: "user", content: "分析这份财报" }],
  plan: [],
  currentStep: 0,
  toolResults: []
});

console.log(result.finalAnswer);
```

```typescript
// 使用 Dexie.js（IndexedDB 封装）
import Dexie from 'dexie';

class PDFDatabase extends Dexie {
  pdfs: Dexie.Table<PDFFile, string>;
  conversations: Dexie.Table<Conversation, string>;
  embeddings: Dexie.Table<Embedding, string>;

  constructor() {
    super('PDFChatDB');
    
    this.version(1).stores({
      pdfs: 'id, fileName, uploadedAt',
      conversations: 'id, pdfId, createdAt',
      embeddings: 'pdfId, chunkIndex'
    });
  }
}

const db = new PDFDatabase();

// 存储 PDF
await db.pdfs.add({
  id: pdfId,
  fileName: 'document.pdf',
  binaryData: arrayBuffer,
  uploadedAt: new Date()
});

// 离线访问
const pdf = await db.pdfs.get(pdfId);
if (pdf) {
  // 使用缓存的 PDF
  displayPDF(pdf.binaryData);
}
```

```
│  ├─ 风险：功能不符合需求
│  ├─ 缓解：用户调研、快速迭代
│  └─ 监控：用户反馈收集
│
└─ 竞争压力
   ├─ 风险：竞品快速发展
   ├─ 缓解：差异化定位、快速创新
   └─ 监控：竞品分析
```

### 结语

让我总结一下我们的 TODO List：

**核心思路：**
1. 先做 Agent 升级（质变）
2. 再做用户系统（规模化）
3. 然后做体验优化（精细化）
4. 最后做商业化（变现）

**时间规划：**
- 6 个月完成核心功能
- 持续迭代优化
- 根据用户反馈调整

**成功关键：**
- 专注核心价值（Agent 能力）
- 快速迭代验证
- 数据驱动决策
- 用户反馈优先

**最重要的：**
不要试图一次做完所有事情，而是：
1. 做好一件事
2. 验证效果
3. 再做下一件

这就是我们的发展路线图！

## Q&A 环节

欢迎大家提问！

常见问题：

**Q1: Agent 会不会太复杂，用户学不会？**
A: 不会。Agent 的复杂性在后端，用户界面反而更简单 - 用户只需要说"做什么"，不需要说"怎么做"。

**Q2: 开发 Agent 需要多少成本？**
A: 主要是 AI API 成本。通过缓存和优化，单用户月成本可控制在 $1-5。

**Q3: 为什么不直接做商业化？**
A: 因为产品价值还不够。先把 Agent 做好，有了核心竞争力，商业化才有意义。

**Q4: 6 个月时间够吗？**
A: 够完成核心功能。但产品永远在迭代，6 个月是 MVP，不是终点。

**Q5: 如果竞品也做 Agent 怎么办？**
A: 欢迎竞争！市场足够大，关键是做出差异化。我们的优势是专注 PDF 场景的深度优化。
