📊 数据流程

1
📤 用户上传 PDF
文件被保存到 /tmp 目录，创建 PDF 记录（状态：PENDING）
2
🔍 PDF 解析
pdf2json 提取文本，失败则使用 OCR 识别
3
✂️ 文本分块 & 向量化
按语义切分文本，调用 AI API 生成向量嵌入
4
💾 存储到 Redis
PDF 元数据、文本 chunks、向量 embeddings 全部存储
5
💬 用户提问
从 Redis 恢复数据，向量搜索，AI 生成回答

🔴 Redis 数据结构

R
Upstash Redis 数据存储
pdf:{id}
PDF 元数据
文件名、页数、状态
vector:{id}
文本 Chunks
分块后的文本
vector:{id}:embeddings
向量 Embeddings
避免重新计算

💻 核心代码示例

// Redis 存储实现
import { Redis } from "@upstash/redis";

const redis = new Redis({
  url: process.env.KV_REST_API_URL,
  token: process.env.KV_REST_API_TOKEN,
});

// 存储 PDF 数据（自动序列化）
await redis.set(`pdf:${pdfId}`, pdfData, {
  ex: 60 * 60 * 24 * 7 // 7天过期
});

// 获取 PDF 数据（自动反序列化）
const data = await redis.get(`pdf:${pdfId}`);
// 兼容性处理
const parsed = typeof data === 'string'
  ? JSON.parse(data)
  : data;

🎯 核心挑战与解决方案

挑战 解决方案 实现
内存变量不持久 Redis 主存储 所有数据存入 Redis，7天过期
后台任务不执行 同步解析 在请求中完成整个解析流程
函数超时限制 多级超时控制 PDF 解析 7秒，OCR 5秒
向量重新计算 缓存 Embeddings 存储向量到 Redis，恢复时复用
@vercel/kv 弃用 迁移到 Upstash 支持 Vercel 集成环境变量
