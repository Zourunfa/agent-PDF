📊 数据流程

1
📤 用户上传 PDF
文件被保存到 /tmp 目录，创建 PDF 记录（状态：PENDING）
2
🔍 PDF 解析
pdf2json 提取文本，失败则使用 OCR 识别
3
✂️ 文本分块 & 向量化
按语义切分文本，调用 AI API 生成向量嵌入
4
💾 存储到 Redis
PDF 元数据、文本 chunks、向量 embeddings 全部存储
5
💬 用户提问
从 Redis 恢复数据，向量搜索，AI 生成回答

💻 核心代码示例

// Redis 存储实现
import { Redis } from "@upstash/redis";

const redis = new Redis({
  url: process.env.KV_REST_API_URL,
  token: process.env.KV_REST_API_TOKEN,
});

// 存储 PDF 数据（自动序列化）
await redis.set(`pdf:${pdfId}`, pdfData, {
  ex: 60 * 60 * 24 * 7 // 7天过期
});

// 获取 PDF 数据（自动反序列化）
const data = await redis.get(`pdf:${pdfId}`);
// 兼容性处理
const parsed = typeof data === 'string'
  ? JSON.parse(data)
  : data;

🎯 核心挑战与解决方案

挑战 解决方案 实现
内存变量不持久 Redis 主存储 所有数据存入 Redis，7天过期
后台任务不执行 同步解析 在请求中完成整个解析流程
函数超时限制 多级超时控制 PDF 解析 7秒，OCR 5秒
向量重新计算 缓存 Embeddings 存储向量到 Redis，恢复时复用
@vercel/kv 弃用 迁移到 Upstash 支持 Vercel 集成环境变量

🤖 什么是 AI Agent 应用

从 RAG 到 Agent 的进化

AI Agent 不仅能回答问题，更能主动规划、使用工具、完成复杂任务。 这是从被动响应到主动执行的质变。

💬 Chatbot

最基础的形态

  • ❌ 被动响应
  • ❌ 单轮对话
  • ❌ 无工具调用
  • ❌ 容易产生幻觉

🔍 RAG

当前状态

  • ⚠️ 被动响应
  • ✅ 多轮对话
  • ⚠️ 单一工具
  • ✅ 准确性高

🤖 AI Agent

升级目标

  • ✅ 主动规划
  • ✅ 多步推理
  • ✅ 多工具协作
  • ✅ 自主决策

Agent 的四大核心能力

📋

1. 自主规划

将复杂任务分解成可执行的步骤 

任务分解：
├─ 1. 了解会议主题
├─ 2. 搜索背景资料
├─ 3. 提取关键数据
├─ 4. 生成会议议程
└─ 5. 准备演示文稿
🔧

2. 工具调用

调用各种工具完成具体任务 

工具箱：
• search_pdf - 搜索文档
• calculator - 数学计算
• web_search - 网络搜索
• generate_chart - 生成图表
• send_email - 发送邮件
🔄

3. ReAct 循环

思考 → 行动 → 观察 → 反思 

💭 Thought: 需要提取数据
🔧 Action: search_pdf()
👁️ Observation: 找到数据
🤔 Reflection: 继续下一步
🧠

4. 记忆系统

记住对话、偏好和中间结果 

记忆类型：
• 短期记忆 - 当前对话
• 长期记忆 - 用户偏好
• 工作记忆 - 中间结果
