🗑️ 问题1：删除用户后数据库不一致

❌ 问题现象：
  • 后台删除用户后，auth.users 仍有数据
  • user_profiles 表为空
  • 登录接口返回"用户不存在"
删除用户
→
删除 user_profiles
→
找不到 user.email
→
❌ auth.users 未删除
✅ 解决方案：

修改删除顺序，先清理关联数据，再删除 auth.users

删除用户
→
清理关联数据
→
删除 user_profiles
→
✅ 删除 auth.users

关键代码修改

// ❌ 错误的删除顺序
await supabase.from('user_profiles').delete().eq('id', userId);
const targetUser = users.find(u => u.email === user.email); // user 已删除！

// ✅ 正确的删除顺序
// 1. 使用 RPC 清理所有关联数据
await supabase.rpc('admin_delete_user_profile', {
  target_user_id: userId
});

// 2. 删除 auth.users
await supabase.auth.admin.deleteUser(userId);

完整删除顺序

  • 1
    清理 user_security_log（安全日志）
  • 2
    清理 quota_usage 和 quota_operations（配额）
  • 3
    清理 conversation_messages（对话消息）
  • 4
    清理 pdf_conversations 和 user_pdfs（PDF数据）
  • 5
    清理 user_sessions（会话）
  • 6
    清理 user_profiles（用户资料）
  • 7
    删除 auth.users（认证用户）

💾 问题2：Next.js 缓存导致用户列表为空

❌ 问题现象：
  • 注册成功，数据库有用户
  • 但后台 API 返回空数组
  • 直接脚本查询有用户
注册用户
→
数据库有用户
→
Next.js API 返回空
→
❌ 列表为空
✅ 根本原因：

Next.js 在构建时缓存了环境变量，运行时使用过期的 Service Role Key

清理缓存步骤

  • 1
    停止开发服务器：pkill -9 -f "next"
  • 2
    删除缓存：rm -rf .next .turbo
  • 3
    重启服务器：npm run dev

快速清理命令

# 一键清理
lsof -ti:3000 | xargs kill -9 && rm -rf .next && npm run dev

# 或使用脚本
./deep-clean.sh

验证缓存是否清理

运行以下命令验证：

curl http://localhost:3000/api/admin/debug-list-users \
  -H "X-Admin-Token: YOUR_TOKEN" | jq '.data'
⚠️ 重要：每次修改 .env.local 后必须清理缓存！

📧 问题3：邮件发送 IPv6 错误

❌ 错误信息： 
Error sending verification email:
connect EHOSTUNREACH 2404:6800:4008:c1b::6c:587
                      ^^^^^^^^^^^^^^^^^^^^^^^
                      IPv6 地址
Nodemailer
→
尝试 IPv6
→
网络不支持
→
❌ EHOSTUNREACH
✅ 解决方案：强制使用 IPv4
项目 修改前 修改后 DNS 设置 默认 dns.setDefaultResultOrder('ipv4first') Socket 配置 无 family: 4 端口 587 (TLS) 465 (SSL) 超时 10秒 15秒

修改后的代码

import dns from 'dns';

// 强制使用 IPv4
dns.setDefaultResultOrder('ipv4first');

transporter = nodemailer.createTransport({
  host: 'smtp.gmail.com',
  port: 465,      // ✅ 改用 SSL 端口
  secure: true,   // ✅ 使用 SSL
  auth: {
    user: process.env.EMAIL_USER,
    pass: process.env.EMAIL_PASSWORD,
  },
  tls: {
    rejectUnauthorized: false,
    minVersion: 'TLSv1.2'
  },
  family: 4,  // ✅ 强制 IPv4
  connectionTimeout: 15000,
  greetingTimeout: 10000,
  socketTimeout: 15000,
});
