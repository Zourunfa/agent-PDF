💡 为什么重要

向量嵌入和 RAG 技术是构建 AI 应用的核心。它们让 AI 能够"理解"文本语义，而不仅仅是关键词匹配。

🔢 什么是向量嵌入？

📝 文本
→
🤖 AI 模型
→
🔢 向量

📝 文本表示

"猫是一种可爱的动物"
"狗是人类的朋友"

🔢 向量表示（简化）

"猫" → [0.8, 0.1, 0.9, 0.2, ...]
"狗" → [0.7, 0.1, 0.8, 0.3, ...]
"汽车" → [0.1, 0.9, 0.2, 0.1, ...]

📊 关键特性

  • 语义相近 — 相似概念的向量距离更近
  • 固定维度 — 通常为 384/768/1536 维
  • 可计算 — 支持余弦相似度等计算

💡 为什么这很重要？

通过向量相似度，我们可以实现"语义搜索"——即使关键词不匹配，也能找到语义相关的内容。这正是 AI 搜索引擎的核心原理。

🗄️ 向量数据库原理

1️⃣ 文档分块
将长文本切分成小块（chunk），每块包含完整语义
↓
2️⃣ 生成嵌入
使用 Embedding 模型将每个文本块转换为向量
↓
3️⃣ 存储索引
将向量和原文一起存储，建立索引结构
↓
4️⃣ 相似度搜索
查询时找出与问题向量最接近的文本块

🤖 什么是 RAG？

检索增强生成 (Retrieval-Augmented Generation)

❓ 用户问题
→
🔍 向量检索
→
📚 相关文档
→
🤖 AI 生成
→
✅ 准确回答

❌ 没有 RAG

  • AI 只能回答训练数据中的知识
  • 无法访问你的私有文档
  • 可能产生"幻觉"（胡说八道）
  • 知识截止于训练时间

✅ 使用 RAG

  • AI 可以访问你的文档库
  • 回答基于真实文档内容
  • 可以引用原文来源
  • 支持实时更新知识
🎤 演讲提示

RAG 是当前 AI 应用的主流架构。它结合了检索系统的准确性和生成模型的自然性，是构建知识问答系统的最佳实践。

💻 LangChain 实现示例

// 1. 创建向量存储
import { SavedVectorStore } from "@langchain/community/vectorstores/in_memory";
import { OpenAIEmbeddings } from "@langchain/openai";

// 2. 初始化嵌入模型
const embeddings = new OpenAIEmbeddings({
  modelName: "text-embedding-3-small"
});

// 3. 创建向量存储（从文本）
const vectorStore = await SavedVectorStore.fromTexts(
  ["PDF 文档内容块 1", "PDF 文档内容块 2", ...],
  ["metadata1", "metadata2", ...],
  embeddings
);

// 4. 相似度搜索
const results = await vectorStore.similaritySearch("用户的问题", 3);
// 返回最相关的 3 个文本块

🔑 关键点

  • 嵌入模型 — 将文本转换为向量
  • 向量存储 — 存储和检索向量
  • 相似度搜索 — 找出最相关的内容
  • 元数据 — 存储文档的附加信息

⚡ 性能优化技巧

💾 向量缓存

避免重复计算

  • 将向量嵌入存储到 Redis
  • 下次直接读取，无需重新计算
  • 节省 API 调用成本

📝 文档分块

优化检索精度

  • 合理的 chunk 大小（500-1000 token）
  • 保留语义完整性
  • 添加重叠区域避免截断

🔍 混合检索

结合多种方法

  • 向量检索 + 关键词检索
  • 重排序（Rerank）优化结果
  • 根据场景调整 k 值
📊 实际效果

通过向量缓存，我们的项目将 PDF 解析后的向量嵌入存储在 Redis 中，避免了每次对话都重新计算，节省了约 70% 的 API 调用成本。

强调 Agent 与 RAG 的本质区别：从被动响应到主动执行。Agent 能够自主规划任务、选择工具、多步推理，这是质的飞跃。

🔧 Agent 的四大核心能力

📋 1. 自主规划

将复杂任务分解成可执行的步骤

任务分解： ├─ 1. 了解会议主题 ├─ 2. 搜索背景资料 ├─ 3. 提取关键数据 ├─ 4. 生成会议议程 └─ 5. 准备演示文稿

🔧 2. 工具调用

调用各种工具完成具体任务

工具箱： • search_pdf - 搜索文档 • calculator - 数学计算 • web_search - 网络搜索 • generate_chart - 生成图表 • send_email - 发送邮件

🔄 3. ReAct 循环

思考 → 行动 → 观察 → 反思

💭 Thought: 需要提取数据 🔧 Action: search_pdf() 👁️ Observation: 找到数据 🤔 Reflection: 继续下一步

🧠 4. 记忆系统

记住对话、偏好和中间结果

记忆类型： • 短期记忆 - 当前对话 • 长期记忆 - 用户偏好 • 工作记忆 - 中间结果

🔄 ReAct 推理循环案例

任务：分析合同的法律风险

第 1 轮循环

💭 Thought: 需要提取合同的关键条款 🔧 Action: search_pdf("违约条款 赔偿责任") 👁️ Observation: 找到 3 个关键条款

第 2 轮循环

第 3 轮循环

💭 Thought: 可以生成风险报告了 🔧 Action: generate_report() 🤔 Reflection: 任务完成 ✅ Final Answer: 生成完整的风险分析报告

🚀 从 RAG 到 Agent 的升级

当前 RAG

async function chat(question) { // 1. 检索文档 const docs = await search(question); // 2. 生成回答 const answer = await ai.chat({ context: docs, question }); return answer; }

❌ 只能回答问题
❌ 不能执行任务
❌ 无法使用多种工具

升级 Agent

async function agent(task) { // 1. 规划任务 const plan = await planner.create(task); // 2. 执行计划（ReAct 循环） for (const step of plan) { const thought = await think(step); const tool = await selectTool(thought); const result = await tool.execute(); if (await reflect(result)) break; } return synthesize(results); }

✅ 可以执行任务
✅ 支持多步推理
✅ 可以使用多种工具

📝 第五部分总结

🎯 核心思路

  • 先做 Agent 升级 — 质变，提升核心竞争力
  • 再做用户系统 — 规模化，支持多用户
  • 然后做体验优化 — 精细化，提升用户满意度
  • 最后做商业化 — 变现，实现商业价值

⚠️ 重要原则

不要试图一次做完所有事情，而是：
1️⃣ 做好一件事
2️⃣ 验证效果
3️⃣ 再做下一件

专注核心价值，快速迭代验证，数据驱动决策
