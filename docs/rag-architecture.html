🎯 什么是 RAG？

📖
定义

RAG (Retrieval-Augmented Generation) 是一种结合了检索系统和生成模型的AI架构。它通过从外部知识库中检索相关信息，然后利用这些信息来增强大语言模型（LLM）的生成能力。

🏗️ RAG 架构详细流程

完整处理流程

1
📄 文档处理
将文档切分成小块（chunk），每个chunk包含完整的语义单元
2
🔢 向量化
使用 Embedding 模型将每个文本块转换为高维向量
3
💾 存储索引
将向量和原文一起存入向量数据库，建立索引结构
4
❓ 问题向量化
用户提问时，将问题也转换为向量
5
🔍 相似度检索
计算问题向量与文档向量的相似度，找出最相关的chunk
6
📝 Prompt 构建
将检索到的相关内容作为上下文，构建完整的 Prompt
7
🤖 LLM 生成
将增强后的 Prompt 发送给 LLM，生成基于上下文的回答
8
✅ 返回结果
将 LLM 的回答返回给用户，并可附加引用来源

⚖️ 传统 LLM vs RAG

❌
传统 LLM（无 RAG）
  • 只能回答训练数据中的知识
  • 无法访问私有文档或实时信息
  • 容易产生"幻觉"（编造答案）
  • 知识截止于模型训练时间
  • 无法引用信息来源
  • 更新知识需要重新训练模型
  • 可能在专业领域表现不佳
✅
RAG 增强 LLM
  • 可以访问外部知识库
  • 支持私有文档和实时信息
  • 基于真实内容，减少幻觉
  • 知识库可随时更新
  • 可以引用原文来源
  • 无需重新训练即可更新知识
  • 在专业领域表现优异

🔍 向量检索详解

向量检索原理

1️⃣ 文本向量化
将文本块通过 Embedding 模型转换为高维向量
例如："人工智能很棒" → [0.2, 0.8, 0.1, 0.9, ...] (1536维)
↓
2️⃣ 问题向量化
用户提问同样转换为向量
例如："AI 好用吗？" → [0.3, 0.7, 0.2, 0.8, ...]
↓
3️⃣ 相似度计算
计算问题向量与所有文档向量的相似度
常用方法：余弦相似度 (Cosine Similarity)
↓
4️⃣ Top-K 检索
返回相似度最高的 K 个文本块
例如：返回最相关的前 3 个文档片段
📐
余弦相似度公式
similarity = cos(θ) = (A · B) / (||A|| × ||B||)
其中 A 和 B 是两个向量，||A|| 和 ||B|| 是它们的模长

取值范围： -1 到 1

含义： 值越接近 1，表示两个向量越相似（语义越相近）

优势： 不受向量长度影响，只关注方向

📝 Prompt 构建详解

📋
RAG Prompt 模板
你是一个专业的AI助手。请根据以下参考信息回答用户的问题。 【参考信息】 {retrieved_context} --- 【用户问题】 {user_query} --- 【要求】 1. 主要基于参考信息回答问题 2. 如果参考信息中没有相关内容，请明确说明 3. 回答要准确、简洁、有条理 4. 可以适当引用参考信息中的具体内容 【回答】：

Prompt 构建步骤

1
系统提示词
定义 AI 的角色和行为准则，设定回答风格
2
检索上下文
插入从向量数据库检索到的相关文档片段
3
用户问题
插入用户的原始问题
4
约束条件
添加回答要求，如"基于参考信息"、"引用来源"等
💡
Prompt 优化技巧
  • 1. 控制上下文长度：根据模型的上下文窗口限制，合理选择检索的文档数量
  • 2. 添加元数据：在检索结果中包含文档来源、日期等信息，便于追溯
  • 3. 明确拒绝回答：指示 AI 当上下文不足时明确说明，而非编造
  • 4. 引用格式：要求 AI 在回答中标注信息来源
  • 5. 思维链：对于复杂问题，可要求 AI 先分析再回答

⚙️ RAG 核心组件

📄
文档加载器 (Document Loader)

功能：从各种来源加载文档

支持格式：PDF、Word、TXT、Markdown、HTML 等

常用工具：LangChain Document Loaders

示例：PDFLoader, TextLoader, WebLoader

✂️
文本分割器 (Text Splitter)

功能：将长文档切分成适合处理的文本块

切分策略：按字符、句子、段落、语义

关键参数：chunk_size, chunk_overlap

常用工具：RecursiveCharacterTextSplitter

🔢
嵌入模型 (Embeddings)

功能：将文本转换为向量表示

常用模型：OpenAI, Cohere, HuggingFace

向量维度：384/768/1536 维

成本考虑：可缓存结果避免重复计算

🗄️
向量数据库 (Vector Store)

功能：高效存储和检索向量数据

常用选择：Pinecone, Weaviate, Chroma, Qdrant

开源方案：FAISS, Milvus, ChromaDB

关键能力：相似度搜索、元数据过滤

🔍
检索器 (Retriever)

功能：根据查询检索相关文档

检索策略：相似度、MMR、混合检索

参数调整：k (返回数量), score_threshold

高级功能：重排序 (Reranking)

🤖
大语言模型 (LLM)

功能：基于上下文生成回答

常用模型：GPT-4, Claude, 通义千问

选择因素：性能、成本、语言支持

部署方式：API 调用或本地部署
