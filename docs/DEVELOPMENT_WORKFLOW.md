## 概述

本文档定义了从**需求提出**到**功能上线**的完整开发流程，适用于：

- 新功能开发
- Bug 修复
- 技术重构
- 性能优化

### 核心原则

| 原则 | 说明 |
|------|------|
| **需求先行** | 先理解要做什么，再考虑怎么做 |
| **文档驱动** | 重要决策必须文档化 |
| **小步快跑** | 大需求拆成小任务，逐步交付 |
| **持续验证** | 每个阶段都要验证是否符合预期 |

```typescript
// ✅ 好的代码
export class UserService {
  constructor(
    private userRepo: UserRepository,
    private emailService: EmailService
  ) {}

  async createUser(data: CreateUserDTO): Promise<User> {
    // 1. 验证数据
    this.validateUserData(data);

    // 2. 检查邮箱是否已存在
    const exists = await this.userRepo.findByEmail(data.email);
    if (exists) {
      throw new ValidationError('邮箱已被使用');
    }

    // 3. 创建用户
    const user = await this.userRepo.create(data);

    // 4. 发送欢迎邮件
    await this.emailService.sendWelcome(user.email);

    return user;
  }

  private validateUserData(data: CreateUserDTO): void {
    // 验证逻辑
  }
}
```

### 开发流程

```
1. 拉取最新代码
   git checkout develop && git pull

2. 创建功能分支
   git checkout -b feature/xxx

3. 开发 + 提交
   git add . && git commit -m "feat: xxx"

4. 推送分支
   git push -u origin feature/xxx

5. 创建 PR
   在 GitHub 上创建 Pull Request

6. Code Review
   团队成员审核代码

7. 合并到 develop
   Review 通过后合并

8. 删除分支
   git branch -d feature/xxx
```

## 实战案例

### 案例：添加"用户头像上传"功能

#### 第一阶段：需求定义

```markdown
# 用户头像上传需求规格

## 背景
用户希望能在个人资料页面上传自定义头像，增强个性化体验。

## 用户故事
作为 用户，我希望 上传自定义头像，以便 展示个人形象。

## 验收标准
- Given 用户已登录, When 用户上传图片, Then 头像更新成功
- Given 用户上传大于 5MB 的图片, When 提交, Then 显示错误提示
- Given 用户上传非图片文件, When 提交, Then 显示格式错误

## 功能需求
| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 上传图片 | P0 | 支持 JPG/PNG，最大 5MB |
| 裁剪图片 | P1 | 支持正方形裁剪 |
| 预览效果 | P1 | 上传前预览 |

## 接口需求
| 接口 | 方法 | 描述 |
|------|------|------|
| /api/user/avatar | POST | 上传头像 |
| /api/user/avatar | DELETE | 删除头像 |
```

#### 第二阶段：技术设计

```markdown
# 用户头像上传技术设计

## 架构图
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  前端组件   │────>│  API 接口   │────>│  Blob 存储  │
│  AvatarUpload│     │ /api/avatar │     │  Vercel     │
└─────────────┘     └─────────────┘     └─────────────┘
                            │
                            ▼
                     ┌─────────────┐
                     │  PostgreSQL │
                     │  user_pdfs  │
                     └─────────────┘
```

## API 设计
POST /api/user/avatar
Request: FormData { file: File }
Response: { success: true, data: { avatarUrl: string } }

## 存储方案
- 使用 Vercel Blob Storage
- 路径: avatars/{userId}/{timestamp}.jpg
- 最大大小: 5MB
- 支持格式: image/jpeg, image/png
```
