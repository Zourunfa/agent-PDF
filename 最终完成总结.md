## 🔄 完整工作流程

### 流程 1: 上传 PDF 并进行对话

```
用户上传 PDF
    ↓
上传 API 保存 PDF 信息到 user_pdfs 表
    ↓
创建对话记录到 pdf_conversations 表
    ↓
用户提问
    ↓
聊天 API 处理问题
    ↓
AI 生成回答
    ↓
保存用户问题到 conversation_messages 表
    ↓
保存 AI 回答到 conversation_messages 表
    ↓
触发器自动更新 pdf_conversations 的统计信息
    ↓
用户看到 AI 回答
```

### 流程 2: 刷新页面后查看 PDF 列表

```
用户刷新页面
    ↓
PDFList 组件自动调用 GET /api/pdfs/list
    ↓
从数据库获取用户的所有 PDF
    ↓
显示 PDF 列表
    ├─ 文件名
    ├─ 文件大小
    ├─ 页数
    ├─ 对话数量 ← 从 pdf_conversations 表获取
    └─ 最后对话时间
    ↓
用户可以看到之前上传的 PDF
```

### 流程 3: 点开 PDF 查看历史对话

```
用户点击 PDF 列表中的某个 PDF
    ↓
ConversationHistory 组件自动调用 GET /api/pdfs/{id}/conversations
    ↓
从数据库获取该 PDF 的所有对话消息
    ↓
显示对话历史
    ├─ 用户问题 (蓝色气泡)
    ├─ AI 回答 (绿色气泡)
    ├─ 消息时间戳
    ├─ Token 数量
    ├─ 处理时间
    └─ 对话统计信息
    ↓
用户可以看到完整的对话历史
```

### 流程 4: 继续对话

```
用户在聊天框输入新问题
    ↓
用户点击发送
    ↓
调用 POST /api/chat
    ↓
聊天 API 处理问题
    ↓
AI 生成回答
    ↓
保存新的对话消息到数据库
    ├─ 保存用户新问题
    └─ 保存 AI 新回答
    ↓
用户看到新的 AI 回答
    ↓
对话历史自动更新
```

## 📊 数据持久化

### 数据存储位置

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| PDF 文件信息 | `user_pdfs` 表 | 文件名、大小、页数等 |
| 对话记录 | `pdf_conversations` 表 | 对话统计信息 |
| 对话消息 | `conversation_messages` 表 | 用户问题和 AI 回答 |

### 数据流向

```
上传 PDF
    ↓
INSERT INTO user_pdfs
    ↓
INSERT INTO pdf_conversations
    ↓
用户提问
    ↓
INSERT INTO conversation_messages (role='user')
    ↓
INSERT INTO conversation_messages (role='assistant')
    ↓
触发器 UPDATE pdf_conversations
    ↓
刷新页面
    ↓
SELECT FROM user_pdfs + pdf_conversations
    ↓
显示 PDF 列表
    ↓
点击 PDF
    ↓
SELECT FROM conversation_messages
    ↓
显示对话历史
```
