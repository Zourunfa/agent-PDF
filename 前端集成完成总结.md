## 📝 具体改动

### 1️⃣ 修改 PDFList 组件
**文件**: `src/components/pdf/PDFList.tsx`

**改动内容**:
```typescript
// 添加了 useEffect 钩子
useEffect(() => {
  const fetchPDFList = async () => {
    const response = await fetch('/api/pdfs/list?limit=50&offset=0&sortBy=uploadedAt&sortOrder=desc');
    const data = await response.json();
    if (data.success && data.data.pdfs) {
      setApiPdfs(data.data.pdfs);
    }
  };
  fetchPDFList();
}, []);
```

**效果**:
- ✅ 页面加载时自动调用 `/api/pdfs/list` 接口
- ✅ 获取 PDF 列表和对话统计信息
- ✅ 显示在左侧 PDF 列表面板

### 2️⃣ 新建 ConversationHistory 组件
**文件**: `src/components/chat/ConversationHistory.tsx`

**功能**:
- ✅ 显示选中 PDF 的对话历史
- ✅ 自动调用 `/api/pdfs/{id}/conversations` 接口
- ✅ 显示用户消息和 AI 回复
- ✅ 显示消息统计信息

**效果**:
- 用户消息显示为蓝色气泡
- AI 回复显示为绿色气泡
- 显示消息时间、token 数、处理时间

### 3️⃣ 修改 AppLayout 组件
**文件**: `src/components/layout/AppLayout.tsx`

**改动内容**:

- 标签页 1: "对话" - 显示聊天界面
- 标签页 2: "历史" - 显示对话历史

**效果**:
- 用户可以在两个标签页之间切换
- 点击"历史"标签页时自动加载对话历史

## 🔄 现在的工作流程

### 页面加载时
```
1. 打开应用
2. PDFList 组件挂载
3. useEffect 触发
4. 调用 GET /api/pdfs/list
5. 获取 PDF 列表
6. 显示在左侧面板
```

### 选择 PDF 时
```
1. 点击 PDF 列表中的某个 PDF
2. activePdfId 更新
3. ConversationHistory 的 useEffect 触发
4. 调用 GET /api/pdfs/{id}/conversations
5. 获取对话历史
6. 显示在右侧面板的"历史"标签页
```

## 📊 API 调用时机

| 事件 | API 端点 | 何时调用 |
|------|---------|---------|
| 页面加载 | GET /api/pdfs/list | PDFList 组件挂载时 |
| 选择 PDF | GET /api/pdfs/{id}/conversations | activePdfId 变化时 |
| 删除 PDF | DELETE /api/pdfs/{id} | 用户点击删除按钮时 |

## 🔍 如果有问题

### 问题 1: 没有看到 API 请求
**解决方案**:
1. 检查浏览器开发者工具是否打开
2. 检查 Network 标签页是否显示所有请求
3. 尝试刷新页面 (Ctrl+R)
4. 检查浏览器控制台是否有错误

### 问题 2: API 返回 401 错误
**解决方案**:
1. 检查是否已登录
2. 检查认证令牌是否有效
3. 尝试重新登录

### 问题 3: API 返回 404 错误
**解决方案**:
1. 检查数据库迁移是否完成
2. 检查 PDF 是否存在
3. 检查 PDF 是否属于当前用户

### 问题 4: 对话历史为空
**解决方案**:
1. 检查是否选择了 PDF
2. 检查该 PDF 是否有对话记录
3. 尝试与 PDF 进行对话
